# Rocket Launcher — Project Context

## Tech Stack
- Unity 6 (6000.4.0f1), 2D, C# 10
- Rigidbody2D physics, Coroutines, no DOTS

## Namespace & Conventions
- namespace RocketLauncher / RocketLauncher.Editor
- File naming: kebab-case cho file mới
- [SerializeField] private _camelCase
- Events: OnPascalCase, subscribe trong OnEnable/OnDisable

## Architecture
- Event-driven: Rocket.OnImpact → RoundManager/Camera
- LaunchController = input ONLY (SRP)
- Physics = FixedUpdate, Input = Update, Camera = LateUpdate

- GameObject.Find() runtime — dùng [SerializeField] ref
- Magic numbers — dùng GameConstants
- public MonoBehaviour fields — luôn [SerializeField] private
- Singleton lazy — inject qua scene references

Implementation 🚀 Phase 2 / 6 — Physics

Rocket.cs — physics + event-driven

Assets/Scripts/Rocket/Rocket.cs (excerpt)
public class Rocket : MonoBehaviour {
    /// <summary>Fired when rocket launched via impulse.</summary>
    public event Action OnRocketLaunched;
    public event Action OnTargetHit;
    public event Action<Vector2, bool, float> OnImpact;

    public void Launch(Vector2 direction, float force) {
        float angle = Mathf.Atan2(direction.y, direction.x)
            * Mathf.Rad2Deg + GameConstants.SpriteAngleOffset;
        transform.rotation = Quaternion.Euler(0, 0, angle);

        _rb.bodyType = RigidbodyType2D.Dynamic;
        _rb.AddForce(direction * force, ForceMode2D.Impulse);
        _isFlying = true;
        _maxHeight = transform.position.y;

        if (_trail != null) _trail.StartTrail();
        OnRocketLaunched?.Invoke();
    }
}
  • Kinematic khi idle → Dynamic khi Launch → Kinematic lại khi reset
  • Event-driven: 0 dependency với RoundManager / Camera — chỉ publish events
  • SpriteAngleOffset = -90f trong GameConstants (sprites point UP)
Implementation 🚀 Phase 2 / 6 — Architecture

Event-driven architecture — loose coupling

Subscribe pattern (trong OnEnable / OnDisable)
// RoundManager.cs
private void OnEnable() {
    _rocket.OnRocketLaunched += HandleLaunch;
    _rocket.OnTargetHit      += HandleWin;
    _rocket.OnImpact         += HandleImpact;
}
private void OnDisable() {
    _rocket.OnRocketLaunched -= HandleLaunch;
    _rocket.OnTargetHit      -= HandleWin;
    _rocket.OnImpact         -= HandleImpact;
}

// CameraController.cs — cũng subscribe nhưng ĐỘC LẬP
private void OnEnable() {
    _rocket.OnRocketLaunched += StartFollow;
    _rocket.OnImpact += (pos, hit, _) => ReturnToVehicle();
}

Slingshot Input — drag-aim-release

LaunchController (Input ONLY)

void Update() {
  if (Input.GetMouseButtonDown(0))
    TryStartDrag();
  if (_dragging) UpdateDrag();
  if (Input.GetMouseButtonUp(0))
    Release();
}

void Release() {
  Vector2 dir = _vehicle.position - _mouseWorld;
  float force = Mathf.Lerp(
    GameConstants.MinLaunchForce,
    GameConstants.MaxLaunchForce,
    dir.magnitude / MaxDrag);
  _rocket.Launch(dir.normalized, force);
}

Implementation 🚀 Phase 4 / 6 — Camera

Camera — intro pan → follow → return

State machine

INTRO_PAN → at game start pan từ Target → Vehicle (2s) ↓ IDLE → đợi player launch camera đứng yên tại Vehicle ↓ OnRocketLaunched FOLLOW → rocket đang bay smooth follow với lerp 0.1 ↓ OnImpact (hit OR miss) RETURN → về Vehicle (1s) coroutine pan với easeInOut ↓ IDLE (reset, vòng lặp)

DRY helper — PanCoroutine

IEnumerator PanCoroutine(
    Vector3 from, Vector3 to,
    float duration,
    Action onComplete = null) {
  float t = 0;
  while (t < duration) {
    t += Time.deltaTime;
    float k = EaseInOut(t / duration);
    transform.position =
      Vector3.Lerp(from, to, k);
    yield return null;
  }
  onComplete?.Invoke();
}
1 helper → dùng cho cả 3 state transitions. DRY theo đúng KISS.

Obstacles — trajectory solver

Vấn đề: obstacle không được chặn đường bay rocket (không đánh đố player).

AI đọc Unity physics docs + tự derive công thức
// Parabolic trajectory: tìm Y tại X trên quỹ đạo rocket
float TrajectoryYAtX(float x, Vector2 launchPos,
                      Vector2 dir, float velocity) {
    float vx = dir.x * velocity;
    float vy = dir.y * velocity;
    float t  = (x - launchPos.x) / vx;         // thời gian đến X
    float g  = Physics2D.gravity.y;            // -9.81f
    return launchPos.y + vy * t + 0.5f * g * t * t;
}

// Spawn strategy: above safe zone OR below safe zone
bool IsSafeSpawn(Vector2 obstaclePos) {
    float trajY = TrajectoryYAtX(obstaclePos.x, ...);
    return Mathf.Abs(obstaclePos.y - trajY) > SafeBuffer;
}

Ground sprite vẫn bình thường, không bị mask. 🔍 Đọc GroundScorch.cs, Rocket.cs, RoundManager.cs, SceneSetupTool.cs (environment partial)... 📋 Root cause: - GroundScorch.groundRef chỉ wire qua SceneSetupTool - Nếu scene tạo manual (không qua tool) → ref = null - null check silent → skip crater creation, không log error - Dev không biết vì "không có gì crash" 🔧 Fix (commit 1a978b7): 1. Fallback: GameObject.Find("Ground") KHI ref null 2. Chỉ dùng Find làm fallback — runtime vẫn dùng serialized ref 3. Thêm Debug.LogWarning khi fallback kicked in 4. Guard #if UNITY_EDITOR cho LogWarning ✅ 1 file sửa, 5 dòng thêm. Crater hiện lại. Time: ~5 phút (AI đọc 4 files song song). Tự viết: ~30 phút (đọc từng file tuần tự).
Debugging

Các bug subtle AI tự phát hiện

Commits 2290cfd, 747c0ff, d30e532, f1302df — không ai report bug, AI tự audit ra.

🔥 STATIC LEAK
Domain reload corrupt
static field giữ reference GameObject. Editor reload → ref stale. Fix: [RuntimeInitializeOnLoadMethod] reset.
💧 TEXTURE LEAK
GPU memory growth
RuntimeSpriteFactory cache texture nhưng không Destroy ở OnDestroy. Editor reload 10 lần → 10x texture.
🏃 COROUTINE RACE
NullRef on restart
Coroutine chạy tiếp sau scene unload → access destroyed object. Fix: StopAllCoroutines() trong OnDestroy.
🎵 AUDIO PITCH LEAK
Sound méo sau reload
AudioSource.pitch modified không reset. Editor reload giữ pitch cũ. Fix: OnDisable reset về 1.0.
⏰ UNCANCELLED INVOKE
Ghost callbacks
Invoke("AutoRestart", 2f) không cancel khi manual restart. Double invoke. Fix: CancelInvoke().
🔒 NULL GUARD SILENT
Hide real bugs
Null check ?. silent hide misconfiguration. Fix: fail-fast pattern — Assert.IsNotNull ở Awake.
AI đọc Unity lifecycle docs + nhận pattern. Dev senior cũng mất 2-3 năm mới quen hết các corner case này.
