Claude Code là cái gì?

Giải thích chi tiết từng feature. Không marketing. Chỉ cái dùng thực tế trong project Rocket Launcher.

◆ TL;DR

Claude Code là AI agent chạy trong terminal — không phải chatbot. Nó đọc toàn bộ codebase (1M token), tự chạy lệnh bash, tự sửa lỗi, spawn sub-agents, gọi tools qua MCP. Khác Copilot (gợi ý line-by-line) hay ChatGPT (copy-paste) ở chỗ: tự hoàn thành task end-to-end.

🧬 Giải phẫu 5 feature chính

① 1M token context — đọc CẢ codebase

Claude Code load được ~1 triệu tokens vào context (Opus 4.7 extended). Quy đổi: ~2,500 file C# trung bình. Project Rocket Launcher (22 scripts, 2,584 dòng) nhét vừa vặn vài lần.

So sánh:

  • Copilot: chỉ thấy file đang mở + 8-16K token window
  • Cursor: chỉ file + cùng folder (~50K)
  • ChatGPT: bạn phải paste file thủ công
  • Claude Code: tự đọc file nào cần → không miss context

② Tool use — tự chạy lệnh

AI có thể gọi các "tools" (functions) trong mỗi turn:

Read Write Edit Bash Grep Glob TaskCreate WebSearch WebFetch Agent (spawn)

Ví dụ: bạn gõ "fix bug trong Rocket.cs", AI sẽ tự:

  1. Read Rocket.cs
  2. Grep tìm related files
  3. Edit Rocket.cs
  4. Bash: chạy Unity test CLI
  5. Đọc output → nếu fail, Edit lại

③ Self-iteration loop

Khi code compile fail, AI không dừng. Nó:

┌─────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐ │ Generate│ → │ Compile │ → │ Read err │ → │ Fix │ └─────────┘ └──────────┘ └──────────┘ └────┬───┘ ↑ │ └──────────────────────────────────────────────┘ (loop cho đến khi compile OK)

④ Sub-agents (Task tool)

AI chính có thể spawn sub-agent song song, mỗi agent 200K token riêng. Ví dụ trong Rocket Launcher: 1 agent Explore codebase, 1 agent research Unity docs, 1 agent chạy tests — kết quả merge về main context.

Lợi ích: main context không bloat, tác vụ độc lập chạy parallel.

⑤ MCP (Model Context Protocol)

Standard để AI gọi tools bên ngoài — như USB cho AI. Rocket Launcher dùng mcp-unity server: AI gọi get_scene_info, update_component, run_tests trực tiếp vào Unity Editor.

Bước 1 · Cài Claude Code

Hướng dẫn cài trên macOS / Linux / Windows. Từ 0 đến prompt đầu tiên trong <10 phút.

◆ Sau slide này bạn sẽ
  • Cài xong Claude Code CLI
  • Có Anthropic account + billing
  • Chạy được claude trong terminal
  • Gõ prompt đầu tiên và thấy response

① Tạo Anthropic account

  1. Vào console.anthropic.com
  2. Đăng ký bằng email (Google OAuth cũng được)
  3. Verify email
  4. Billing → add credit card (tối thiểu $5 credit để dùng API)
  5. Settings → API Keys → Create Key → copy key, lưu cẩn thận
⚠ Lưu ý billing
  • Nếu dùng Claude Pro ($20/tháng) → không cần API key, chỉ cần đăng nhập bằng email
  • Nếu dùng API (pay-as-you-go) → tạo key, set rate limit để tránh surprise bill
  • Dev mới: Pro $20 là đủ. Chuyển sang API khi biết rõ usage pattern.

② Cài CLI

macOS / Linux

bash
curl -fsSL https://claude.ai/install.sh | bash

Hoặc qua npm (tất cả OS)

bash
npm install -g @anthropic-ai/claude-code

Windows

Dùng WSL2 (Ubuntu) là mượt nhất. Cài WSL xong chạy command Linux phía trên. Native PowerShell:

  1. Hỏi xác nhận tạo file (lần đầu, sau đó whitelist)
  2. Tạo hello.cs với nội dung Main method
  3. Báo "File created"

✓ Checklist

⚠ Lỗi thường gặp
  • "command not found: claude" → thêm ~/.claude/bin vào $PATH (thường installer báo cách thêm, đọc kỹ output)
  • "Authentication failed" → chạy lại claude login, browser phải mở được

  • M1/M2 Mac npm lỗi → dùng nvm install Node 20+ trước

Bước 2 · Project structure Unity

AI code tốt hơn khi project có structure rõ. Slide này dựng layout chuẩn dùng được cho mọi Unity 2D project.

# Rocket Launcher — Project Context

Bạn đang làm việc trên game Unity 2D physics slingshot.
Luôn tuân thủ các quy tắc dưới đây khi sinh hoặc sửa code.

## Tech Stack
- Unity 6 (6000.4.0f1), 2D Built-in Render Pipeline
- C# 10, .NET Standard 2.1
- Rigidbody2D physics, Coroutines (KHÔNG DOTS)

## Namespace
- Runtime: `namespace RocketLauncher`
- Editor: `namespace RocketLauncher.Editor`

## Code Conventions
- Private field: `[SerializeField] private float _launchForce = 10f;`
- Events: `public event Action OnRocketLaunched;`
- Null guards: fail-fast với Assert.IsNotNull ở Awake
- XML docs trên mọi public member

## Architecture Rules
- Event-driven: Rocket → RoundManager/Camera subscribe
- LaunchController = input ONLY
- Physics: FixedUpdate / Input: Update / Camera: LateUpdate
- Magic numbers → GameConstants.cs

## KHÔNG LÀM
- `GameObject.Find()` runtime — dùng `[SerializeField]` ref
- `public` fields trên MonoBehaviour — luôn `[SerializeField] private`
- Singleton lazy `Instance` — register qua scene GameObject

🎯 Tại sao mỗi rule đều có ý nghĩa

RuleTại sao
GameObject.Find() bị cấmSlow (scan Hierarchy), fragile (rename break), silent fail (null không log)
[SerializeField] privateEncapsulation + visible trong Inspector — best of both
Event-driven architectureLoose coupling — thêm subscriber = 0 thay đổi publisher
Physics ở FixedUpdateDeterministic — Update() variable frame rate làm physics giật
Camera ở LateUpdateĐợi tất cả physics/transform settle xong mới move camera → không lag 1 frame
Magic numbers → GameConstantsSửa 1 chỗ, cả codebase update. Test dễ (mock constant). Đọc code dễ (tên semantic).

using System;
using UnityEngine;

namespace RocketLauncher {
    public class Rocket : MonoBehaviour {
        public event Action OnRocketLaunched;
        public event Action OnRocketLanded;
        public event Action OnTargetHit;
        public event Action<Vector2, bool, float> OnImpact;

        [SerializeField] private Rigidbody2D _rb;
        private bool _isFlying;
        private float _maxHeight;

        public void Launch(Vector2 direction, float force) {
            float angle = Mathf.Atan2(direction.y, direction.x)
                * Mathf.Rad2Deg + GameConstants.SpriteAngleOffset;
            transform.rotation = Quaternion.Euler(0f, 0f, angle);
            _rb.bodyType = RigidbodyType2D.Dynamic;
            _rb.AddForce(direction * force, ForceMode2D.Impulse);
            _isFlying = true;
            _maxHeight = transform.position.y;
            OnRocketLaunched?.Invoke();
        }

        private void FixedUpdate() {
            if (!_isFlying) return;
            if (_rb.linearVelocity.sqrMagnitude > MinVelocitySqr) {
                float angle = Mathf.Atan2(
                    _rb.linearVelocity.y, _rb.linearVelocity.x)
                    * Mathf.Rad2Deg + GameConstants.SpriteAngleOffset;
                _rb.MoveRotation(angle);
            }
            _maxHeight = Mathf.Max(_maxHeight, transform.position.y);
        }

        private void OnCollisionEnter2D(Collision2D col) {
            if (!_isFlying) return;
            bool hit = col.collider.CompareTag(TargetTag);
            Vector2 pos = col.GetContact(0).point;
            _isFlying = false;
            if (hit) OnTargetHit?.Invoke();
            else if (col.collider.CompareTag(GroundTag)) OnRocketLanded?.Invoke();
            OnImpact?.Invoke(pos, hit, _maxHeight);
        }
    }
}

🎓 Chi tiết đáng chú ý

① SpriteAngleOffset = -90°

Sprite rocket vẽ hướng UP (Y+). Velocity có thể hướng bất kỳ. atan2(y, x) trả về góc so với X+. Để sprite hướng UP khi vel=(0,1), offset bằng -90°.

② MoveRotation vs transform.rotation

Trong FixedUpdate, dùng _rb.MoveRotation(angle) để physics engine biết — không bypass physics. Ngược lại trong Launch() initial rotate, dùng transform.rotation OK vì chưa Dynamic.

③ sqrMagnitude > MinVelocitySqr

Khi velocity gần 0 (sắp dừng), rotation nhảy lung tung do atan2 nhạy cảm với giá trị nhỏ. Guard bằng threshold.

④ Event pattern

Events public, class không care ai subscribe. Test Rocket độc lập — subscribe trong test, fire event, assert.

⚠ Lần đầu AI sinh sai gì

AI version đầu dùng transform.rotation trong FixedUpdate — không tôn trọng physics. Review agent flag, AI sửa thành MoveRotation trong commit 8452adb:

public class LaunchController : MonoBehaviour {
    [SerializeField] private Rocket _rocket;
    [SerializeField] private Transform _vehicle;
    [SerializeField] private float _maxDragDistance = 5f;

    public event Action<Vector2, float> OnAimChanged;

    private void Update() {
        if (_rocket.IsFlying) return;
        if (Input.GetMouseButtonDown(0)) TryStartDrag();
        if (_dragging && Input.GetMouseButton(0)) UpdateDrag();
        if (_dragging && Input.GetMouseButtonUp(0)) Release();
    }

    private void Release() {
        var drag = GetDragVector();
        if (drag.magnitude < 0.5f) return;  // cancel too small
        var dir = -drag.normalized;
        var force = Mathf.Lerp(
            GameConstants.MinLaunchForce,
            GameConstants.MaxLaunchForce,
            drag.magnitude / _maxDragDistance);
        _rocket.Launch(dir, force);
    }
}

private IEnumerator PanCoroutine(Vector3 from, Vector3 to,
    float duration, Action onComplete = null) {
    float t = 0f;
    while (t < duration) {
        t += Time.deltaTime;
        float k = EaseInOutQuad(t / duration);
        transform.position = Vector3.Lerp(from, to, k) + _zOffset;
        yield return null;
    }
    transform.position = to + _zOffset;
    onComplete?.Invoke();
}

    private const int MaxCraters = 20;
    private static readonly Queue<GameObject> _craters = new();

    public static void CreateCrater(Vector2 impactPos, float radius) {
        if (_craters.Count >= MaxCraters) {
            var old = _craters.Dequeue();
            if (old != null) Object.Destroy(old);
        }
        var mask = new GameObject("Crater_SpriteMask");
        mask.transform.position = impactPos;
        var sm = mask.AddComponent<SpriteMask>();
        sm.sprite = RuntimeSpriteFactory.GetCircleSprite();
        _craters.Enqueue(mask);

    public static float GetGroundY(float x) {
        float y = GameConstants.GroundTop;
        foreach (var c in _craters) {
            if (c == null) continue;
            float dx = x - c.transform.position.x;
            float r = c.transform.localScale.x * 0.5f;
            if (Mathf.Abs(dx) < r)
                y -= Mathf.Sqrt(r * r - dx * dx);
        }
        return y;
    }
}

public static AudioClip GenerateStretch() {
    const int sr = 44100;
    const float duration = 0.3f;
    int samples = (int)(sr * duration);
    var data = new float[samples];

    for (int i = 0; i < samples; i++) {
        float t = (float)i / sr;
        float freq = Mathf.Lerp(200f, 800f, t / duration);  // sweep up
        float envelope = Mathf.Exp(-t * 3f);  // decay
        data[i] = Mathf.Sin(2f * Mathf.PI * freq * t) * envelope * 0.3f;
    }

    var clip = AudioClip.Create("stretch_proc", samples, 1, sr, false);
    clip.SetData(data, 0);
    return clip;
}

Dev không viết công thức. AI đọc Physics2D.gravity docs + parabola math tự derive. Unit test xác nhận 11/11 pass.

🎯 Problem

Obstacle spawn random, không được chắn đường bay của rocket lần đầu. Nếu chắn, player miss forever — game frustrating.

🧮 Math derivation

Rocket launch với initial velocity v = direction × force. Physics2D gravity g = -9.81. Trajectory parabolic:

▲ y │ │ ● trajectory: y(t) = y0 + vy·t + ½·g·t² │ . │ . . │. . ● ● launch pos impact

Tại position x, trajectory Y:

t = (x - x0) / vx
y(x) = y0 + vy·t + 0.5·g·t²

// Compute trajectory Y at given X
private float TrajectoryYAtX(float x, Vector2 launchPos,
                              Vector2 dir, float velocity) {
    float vx = dir.x * velocity;
    float vy = dir.y * velocity;
    if (Mathf.Abs(vx) < 0.001f) return launchPos.y; // div zero
    float t = (x - launchPos.x) / vx;
    float g = Physics2D.gravity.y; // -9.81
    return launchPos.y + vy * t + 0.5f * g * t * t;
}

// Safe spawn check
private bool IsSafeSpawn(Vector2 pos, Vector2 launchPos,
                           Vector2 dir, float velocity) {
    float trajY = TrajectoryYAtX(pos.x, launchPos, dir, velocity);
    return Mathf.Abs(pos.y - trajY) > GameConstants.ObstacleSafeBuffer;
}
